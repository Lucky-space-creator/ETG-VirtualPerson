# AI数字人导游智能导览系统 — 产品总体设计文档

---

## 1 绪论

### 1.1 课题研究背景

国家大力推进文旅产业数字化转型，智慧景区建设已成为行业趋势。传统景区导览服务面临讲解资源不足、信息更新滞后、难以满足游客个性化与深度体验需求等痛点。黄金周或旺季时，专业导游供不应求；传统的录音导览设备内容固定，无法与游客互动；冰冷的设备难以提供如真人导游般的亲切感和情感互动；景区管理者难以量化评估导游服务质量。

本课题旨在利用AI数字人技术，构建一个智能、互动、个性化的景区导览服务AI数字人，推动文旅服务的智能化升级。

### 1.2 国内外研究现状

当前数字人技术在文旅领域的应用主要集中在以下几个方向：

- **2D数字人驱动**：以duix-mobile为代表的开源方案，通过预渲染帧序列实现数字人动画，支持实时语音驱动口型同步
- **3D数字人渲染**：基于Three.js、Unity等引擎的3D模型渲染，支持VRM标准格式的数字人模型
- **大语言模型应用**：基于GPT、通义千问等大模型的智能问答系统，结合RAG技术实现精准知识检索
- **多模态交互**：融合语音识别、语音合成、表情驱动的多模态人机交互技术

### 1.3 课题意义

本系统通过AI数字人技术，构建7×24小时在线的"智能导游"，解决景区导游资源紧张问题，提升游客游览体验的沉浸感和满意度，帮助景区管理者数字化运营，降低人力成本，实现服务升级。

### 1.4 文档的内容

本文档详细描述AI数字人导游智能导览系统的需求分析、系统设计、系统实现和系统测试，涵盖游客交互端和管理后台的完整功能。

## 2 需求分析

### 2.1 需求陈述

#### 功能性需求

**游客交互侧：**
1. 多模态交互：支持语音输入和文本输入，数字人能以语音、表情和口型同步的方式进行回答
2. 智能问答与讲解：能够准确回答关于景区历史、文化、景点特色等常见问题
3. 个性化推荐：可根据游客兴趣推荐不同的游览路线和讲解重点

**管理后台侧：**
1. 知识库管理：管理员可上传、更新和维护景区的讲解词、文史资料、常见问题及答案等知识文档
2. 数字人形象管理：可配置数字人的外观、服装、声音等
3. 游客感受度报告：系统能分析交互记录，生成游客关注点分析、情感趋势报告
4. 数据大屏概览：展示当日/本周服务人次、热门问答、游客满意度趋势等核心运营数据

#### 非功能性需求

1. 需明确使用至少1个多模态大模型作为核心AI能力支撑
2. 准确度：对于景区问题的事实性问答的准确率不低于90%
3. 交互响应延迟应控制在合理范围内（语音问答延迟<5秒）
4. 系统稳定，不应出现崩溃或长时间无响应

### 3.3 功能详细设计

#### 3.3.1 数字人渲染与动画

**渲染方案：**
- 使用Three.js在WebView中渲染VRM格式3D模型
- WebGL渲染器支持透明背景，叠加在背景图片上
- VRM模型通过glTF格式加载，支持骨骼动画和表情系统

**动画系统：**
- 基于关键帧的动画系统（vrm-animation.js）
- 预定义16种动作：点头、招手、鞠躬、开心、悲伤、惊讶等
- 支持动作队列和优先级管理
- 使用easeInOutCubic缓动函数实现平滑过渡

**骨骼控制：**
- 上臂旋转：±50度（0.873弧度）自然下垂
- 前臂微弯：-0.3弧度
- 手腕自然弯曲：±0.1弧度
- 待机动画：呼吸浮动、眨眼、轻微摇摆

#### 3.3.2 智能问答与RAG检索

**RAG检索流程：**
1. 用户输入查询文本
2. 向量检索：使用BGE-large-zh模型生成查询向量，与知识库向量进行相似度匹配
3. BM25检索：基于关键词的传统文本检索
4. RRF融合：将向量检索和BM25检索结果通过RRF算法融合
5. 重排序：使用BGE-reranker对融合结果进行重排序

**知识库管理：**
- 支持PDF、DOCX、TXT、MD、HTML格式文档
- 支持三种切割策略：按文档结构、固定大小、语义切割
- 按景区隔离知识库，支持多景区独立管理

#### 3.3.3 语音合成与口型同步

**TTS方案：**
- 使用Edge-TTS进行语音合成
- 支持6种中文语音：晓晓、晓伊、云希、云健、晓辰、云泽
- 长文本自动分段，每段不超过100字符

**口型同步：**
- 基于定时器模拟口型变化
- 多频率混合：快节奏（200ms）、中节奏（400ms）、慢节奏（800ms）
- 平滑过渡：lastMouthValue = lastMouthValue * 0.7 + value * 0.3
- 支持5种口型：aa、ih、ou、ee、oh

#### 3.3.4 游客消费分析

**数据导入：**
- 支持Excel格式消费数据导入
- 自动识别列名：tourist_id、age、gender、attraction_name、total_cost等
- 支持批量导入和数据校验

**分析维度：**
- 消费结构：门票、餐饮、购物、交通、娱乐占比
- 年龄分布：18-25、26-35、36-45、46-55、56岁以上
- 满意度分布：非常满意、满意、一般、不满意
- 消费模式：高消费、购物主导、餐饮主导、家庭出游
- 景点排行：按游客数、消费额排序

### 4.2 智能问答实现

**RAG混合检索：**
```python
class HybridSearcher:
    def hybrid_search(self, query, kb_id, top_k=5):
        # 向量检索
        vector_results = self.vector_search(query, kb_id, top_k * 2)
        # BM25检索
        bm25_results = self.bm25_search(query, top_k * 2)
        # RRF融合
        fused = self.rrf_fusion(vector_results, bm25_results)
        return fused[:top_k]
```

## 结论

本系统成功实现了AI数字人导游智能导览系统的核心功能，包括：

1. **数字人渲染与交互**：基于Three.js + three-vrm实现了3D数字人渲染，支持口型同步、表情控制和多种动作动画，为游客提供沉浸式交互体验。

2. **智能问答系统**：集成RAG检索和大语言模型，实现了基于景区知识库的精准问答，回答准确率高，响应延迟控制在合理范围内。

3. **多模态交互**：支持文本输入、语音输入、语音输出和口型同步，实现了完整的多模态交互链路。
